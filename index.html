Muse Music Player - WearHacks

WearHacks was a hackathon that focused on the use of leading wearable technology. Some devices made available to us were the Nod Ring, the Myo armband and even the Muse Brain Sensing Headband. Out of all the innovative wearable tech available, my team decided to work with the Muse.

With the Muse, we wanted to create something simple and smart. The muse is able to read brainwaves, which means that it can, to an extent, read a person's behaviour or emotion. We decided to make a music player that will automatically generate an ideal playlist for a user based on previous analysis performed while they listened to various tracks.

This is how it worked:

  • A user will listen to each song in the list once.
  • Using the Muse, we will record and determine if the user has enjoyed the song or not.
  • After the user has finished listening to all the songs, the app will take all the songs with the highest scores and add them to the ideal playlist.

    • In order to determine how different brainwaves (alpha, beta, gamma, etc) map to behaviour, we performed a number of tests with an individual wearing the device. We then recorded the muse readings and used Matlab and differential calculus to remove noise and analyze the signals. In the end, we were able to somewhat determine a general idea of what signal patterns map to the emotion "Joy".

    In the end, we didnt have time to complete the project (we realized that we weren't using the correct android library version, so it didn't fully work), but we had gained practical experience in signal analysis, which was still a very valuable asset in my opinion.

    ChallengeHub - BattleHack (See the project here)

    At BattleHack, we wanted to solve a problem that we've experienced. We wanted to create a more efficient and solid way of assessing a developer's skills that will replace the nerve-racking whiteboard interviews. We realized that a lot of very skilled developers may not perform at their best during the pressure of a whiteboard test in an interview. What we created was a way to assess the potential employee using a company's own code base.

    ChallengeHub is a github integrated application that allows for companies to send out "Challenges" with an option of a monetary incentive. These challenges would be a chance to fix a but or add a new feature to the company's open source code. It was created using ReactJS for frontend, and NodeJS and MongoDB as a backend. Braintree was also integrated as the payment system.

    In the perspective of an employer, they would benefit from the ability to test a developer in the environment where they will be working. Being able to see the exact implementation style of a potential developer is a great asset for an employer. Developers would also benefit from having their full skills exposed by showing what they can do in an real development environment.
