Work Experience

Through my career, I've had 7 internships. Professionally, I started out as a front end developer, but I've been more interested in backend work recently. In this section, I will be describing my professional work experience. For a more condensed version, you can see my resume.

Backend Software Engineer - Yelp
September 2017 - December 2017

My second internship at Yelp was focused on backend work.

I mostly worked with data processing on the Transactions team. There were many questions we wanted to answer with our food order data, but they weren't feasible because of the number of slow MySQL database queries that would have to be called to answer those questions. We needed a faster way to provide answers to our users. I worked on a joint project with my mentor called Fast Reorders. We wanted to give useful information to our users about their previous orders, so we utilized Yelp's DataPipeline to take our orders from Kafka and insert them into Elastic Search.

I worked on various message processing applications (Yelp calls these "Spolts") that transforms orders in our Kafka topic to something more useful. For example, I wrote a Spolt to take an order, find the latitude and longitude, and then output the new geolocated order back into a downstream Kafka topic. This would be useful to find reorders near a user's location.

One other thing that I worked on was an auditing batch that would find missing orders from Elastic Search. It would then proceed to insert the missing orders back into Elastic Search.

Tools used: Python, Kafka, Elastic Search, Yelp PaaSTorm, Yelp PaaSTA, Git, Splunk, SignalFX

Fullstack Software Engineer - Yelp
September 2016 - December 2016

At Yelp, I was on the Transactions team.

The Transactions team was responsible for closely connecting users and businesses through delivery and pickup (and other services as well). Third party partners work with us to provide the delivery of the service directly to the customers.

During these four months, I refactored our team's frontend pages into our own frontend service. Yelp's codebase was at one point a single monolithic app, and in the last couple of years, there has been a great effort to move parts of that monolithic app into their own separate service to improve scalability. However, most of our team's frontend pages were still stuck in the main Yelp app. We wanted to have our own service so that we could fully control the deployment and development of our own frontend pages.

The Transactions team had its own development playground called the Platform Playground. It allows us to test our order flow end to end by creating development docker containers and configuring the ports to point to each other instead of the regular development containers. This allowed us to test new changes in multiple services with little effort, and it allowed for simple acceptance testing as well. The Transactions team had never integrated a frontend service into the playground, so I was responsible for this.

This project gave me an opportunity to actually work with every part of the service. I integrated it into the testing suite, wrote docker files, set up dashboards for service monitoring and log ingestion, and refactored the frontend pages.

Tools used: Python, JS, Git, Docker, Splunk, SignalFX, Yelp PaaSTA, SmartStack

Software Engineering Intern - Nexj Systems
May 2015 - August 2015

I was part of the Nexj Health team working on a product called Connected Wellness, a wellness solution using wearable technology.

In order to better understand patients, Connected Wellness allowed "Assessments" to be assigned by a health care professional (HCP) so that they can better evaluate the patient's current health. Assessments are pre- and post- clinical surveys that are given to a patient to increase the efficiency of the appointment. Assessments would be scored to determine what state the individual's health is in, and then added to a group according to the results. I mainly worked on the team responsible for making an those assessments. I also implemented most of the scoring logic for the assessments.

In addition to the assessments, I was able to modify and add mobile responsive features in other parts of the app.

At Nexj Health, I also spent some time fixing server side issues. I used tools such as Kibana, Fiddler and the Node Inspector to analyze data and various http requests. In order to sufficiently test the security of the app, I needed to intercept http requests, modify the request, and resend it in order to try to use the app in ways it wasn't meant to be used.

Tools used: JavaScript, NodeJS, MongoDB, Less, Mercurial, SOA, Vagrant, Kibana, Fiddler
